using your own OpenAI API key

The From text feature turns a word into pixel art using OpenAI's image model. Each person uses their own API key, so you only ever pay for images you generate. The From image feature (converting an uploaded photo) needs no key at all.

1. Create an OpenAI account

Go to platform.openai.com/signup and sign up (or log in if you already have an account).

2. Add billing

Image generation is a paid API. Open Billing and add a payment method / credit. Images are inexpensive — typically a few cents each — and you can set a monthly spend limit to stay in control.

3. Create an API key

  1. Open platform.openai.com/api-keys.
  2. Click Create new secret key.
  3. Copy the key — it starts with sk-. You won't be able to see it again, so paste it somewhere safe.

4. Paste it into PixelMaker

Back on the main page, open the From text tab and click Set API key. Paste your key into the dialog, leave Remember on this device checked to avoid re-entering it next time, and click Save. The status will show Key saved. Then type a subject (e.g. alligator) and click Generate. For security the key is not shown again; use Set API key to replace it or Clear saved to remove it.

Where is my key stored?

If you tick Remember on this device, the key is saved only in your own browser (via localStorage) — it is not stored on the server. When you generate, the key is sent with that single request to this app's backend, which uses it to call OpenAI and then discards it. It is never logged or shared. To remove a saved key, clear the box (or untick the remember option) and generate once, or clear your browser site data.

Troubleshooting

  • "No API key provided" — paste your sk-... key into the key box.
  • "Image generation failed: ... 401" — the key is wrong or revoked; create a new one.
  • "... 429 / insufficient_quota" — add billing/credit to your OpenAI account.
  • Nothing happens — the image tab still works without a key; check your connection for the text tab.
